---
title: Quickstart
subtitle: List your channels and queue your first post in three requests.
---

This walkthrough assumes you have a Supabase access token for a user who is a member
of at least one channel. See [Authentication](./authentication.mdx) for how to obtain one.

<Steps>

### Set your token

```bash
export TEMPO_TOKEN="<your-supabase-access-token>"
export TEMPO_BASE="https://tempoapp.hq/api"
```

### List the channels you can access

```bash
curl "$TEMPO_BASE/channels" \
  -H "Authorization: Bearer $TEMPO_TOKEN"
```

```json
[
  {
    "id": "8f3c...",
    "account_handle": "yourbrand",
    "display_name": "Your Brand",
    "membership_role": "owner"
  }
]
```

### Add a post to the queue

Omit `scheduled_at` and Tempo auto-assigns the next open slot. Pass one to pin the post
to an exact time (you'll get a `409` if that slot is taken).

```bash
curl -X POST "$TEMPO_BASE/posts" \
  -H "Authorization: Bearer $TEMPO_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "channel_id": "8f3c...",
    "text": "Hello from the Tempo API"
  }'
```

</Steps>

## Same call, TypeScript SDK

The SDK below is generated by Fern from the same OpenAPI spec that powers this site,
so it always matches the API.

```typescript
import { TempoClient } from "@tempo/sdk";

const tempo = new TempoClient({ token: process.env.TEMPO_TOKEN });

const channels = await tempo.channels.list();
await tempo.posts.create({
  channelId: channels[0].id,
  text: "Hello from the Tempo API",
});
```
